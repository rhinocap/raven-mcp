### Weekly knowledge-PR pipeline
**What:** Added a GitHub Action that runs weekly (Mondays 07:00 UTC, or manual dispatch), reads open issues labelled `knowledge-request`, and asks Claude Opus 4.7 to draft a matching JSON file in `src/data/`. Each drafted issue gets its own branch + PR for Andrew to review + merge.
- `scripts/draft-knowledge-pr.mjs` — Node script using `@anthropic-ai/sdk` (added as dev dep). Prompt-cached system prompt includes the full CTA pattern + partial laws-of-ux principle as schema examples. Strict JSON plan extraction. Falls back gracefully on error (restores main, deletes branch).
- `.github/workflows/knowledge-pr.yml` — cron + workflow_dispatch. Needs `ANTHROPIC_API_KEY` added to repo secrets. Uses built-in `GITHUB_TOKEN`.
- `.github/ISSUE_TEMPLATE/knowledge-request.yml` — structured form so submitters pass layer/name/summary/evidence in a shape the drafter can consume reliably.
- `scripts/release.sh` + `npm run release` — one-command weekly release: bumps version (patch/minor/major), syncs `manifest.json`, rebuilds `.mcpb`, `npm publish`, commits, tags, pushes. Guards against dirty tree and non-main branch.

**Why:** Andrew wants Raven to get better over time without manual work. LLM-drafted PRs is the chosen path (vs telemetry, vs self-modifying). Claude API on GH Action vs local LLM: API wins 2-3× on realistic throughput since draft quality dominates Andrew's review time. Cost ~$1-5/month at expected volume.
